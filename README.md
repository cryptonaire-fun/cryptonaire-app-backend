# Cryptonaire Backend API

Welcome to the backend repository of **Cryptonaire**, a real-time crypto trivia and gaming application! This repository houses the API that handles user authentication via Solana wallets, AI-powered question generation, user progression, and global leaderboards.

> **Devnet Notice:** This project simulates the full `$SKR` token economy on **Solana devnet**. Because the real SKR token exists only on mainnet, the setup script creates a custom SPL token mint on devnet that mirrors SKR's behavior (6 decimals, same transfer mechanics, on-chain metadata). The entire withdrawal flow — treasury wallet, ATA creation, SPL transfers — is identical to what a mainnet deployment would use; only the network and mint address differ.

## 🚀 Features

- **Wallet-Based Authentication**: Secure login using Solana wallet signatures (`Sign in with Solana`), backed by JWT sessions.
- **AI Question Generation**: Integration with Groq AI to dynamically generate crypto-related questions for players.
- **Progression System**: Track user progress, including points, tokens, and automatic level-ups as they answer questions correctly. Each correct answer awards a fixed **0.1 SKR token** (server-enforced) and increments `questionsAnswered`. Level is calculated from cumulative correct answers using the formula: Level N requires `N × 10` questions to advance (10 to reach Level 2, 20 more for Level 3, etc.).
- **SKR Token Withdrawals**: Users can withdraw their earned `$SKR` tokens on-chain to their Solana wallet. The backend treasury signs and pays for all transfers via SPL token — no SOL required from the user. On devnet, a simulated SKR mint is used; the flow is identical to mainnet.
- **Global Leaderboard**: Real-time ranking of players based on their scores.
- **Robust Security**: Protected routes, JWT verification, rate-limiting, and standard Express security middlewares (Helmet, CORS).

## 🔧 Installation & Setup

1. **Clone the repository:**

   ```bash
   git clone <repository-url>
   cd cryptonaire-app-backend
   ```

2. **Install dependencies:**

   ```bash
   bun install
   ```

3. **Environment Configuration:**
   Create a `.env` file in the root directory based on `.env.example`:

   ```bash
   cp .env.example .env
   ```

   *Required Variables:*
   - `PORT`: (default 4000)
   - `NODE_ENV`: `development` or `production`
   - `MONGODB_URI`: Your MongoDB connection string
   - `JWT_SECRET`: A strong random string for signing JWT tokens
   - `JWT_EXPIRES_IN`: JWT expiration length (e.g., `7d`)
   - `GROQ_API_KEY`: Your Groq AI API key

   *Solana / SKR Variables (for token withdrawals):*
   - `SOLANA_RPC_URL`: Solana RPC endpoint (defaults to `https://api.devnet.solana.com`)
   - `TREASURY_PRIVATE_KEY`: JSON byte-array of the treasury keypair — generated by the setup script below
   - `SKR_MINT_ADDRESS`: Devnet SPL token mint address — generated by the setup script below

4. **Set up the devnet SKR mint (one-time):**

   Since the real `$SKR` token only exists on mainnet, this script creates a simulated equivalent on devnet — a custom SPL token with the same 6-decimal precision and on-chain metadata (`name: "SKR"`, `symbol: "SKR"`). It generates a treasury keypair, mints 1,000,000 tokens to it, and attaches metadata so wallets like Phantom display it correctly. The withdrawal flow is identical to what a mainnet deployment would use.

   ```bash
   bun run scripts/setup-devnet-mint.ts
   ```

   The script will print the treasury public key and pause. Go to [faucet.solana.com](https://faucet.solana.com), paste the address, and airdrop SOL. The script will detect the funding and continue automatically, then print the values to add to your `.env`.

5. **Start the development server:**

   ```bash
   bun run dev
   ```

   The server will start locally at `http://localhost:4000`.

## 🛣️ API Endpoints Summary

### Authentication

- `POST /auth/verify` - Verifies Solana wallet signature and returns a JWT token.

### User

- `GET /user/me` - Retrieves the authenticated user's details.
- `GET /user/me/points` - Retrieves only the user's total points.
- `GET /user/me/tokens` - Retrieves only the user's SKR token balance.
- `GET /user/me/progress` - Retrieves user's level, questions answered, and progression status.
- `PATCH /user/me/username` - Update the user's display name.
- `POST /user/me/withdraw` - Withdraw SKR tokens on-chain to the user's Solana wallet.

### Game

- `POST /game/generate-questions` - Generates a new set of crypto questions using Groq AI.
- `POST /game/add-to-user-points` - Increments user points upon a correct answer. Also automatically awards **0.1 SKR token** and increments `questionsAnswered`, triggering a level-up if the threshold is reached.
- `POST /game/add-to-user-tokens` - Credits a specified amount of SKR tokens to the user.
- `POST /game/add-to-user-both` - Increments user points upon a correct answer and automatically awards **0.1 SKR token**, with level progression applied.

### Leaderboard

- `GET /leaderboard` - Public endpoint to retrieve the global scoreboard.
- `GET /leaderboard/:address` - Retrieve leaderboard position for a specific wallet address.

## 🔒 Security Measures

- Uses **bcryptjs** and **tweetnacl** for cryptographic operations.
- Request payloads validated strictly via **Zod**.
- **express-rate-limit** prevents brute-force attacks.
- **Helmet.js** sets sensible and secure HTTP headers.
